## 4. 四种牌型

```
detect_pattern(cards, skip_clan_bomb=false) 判定优先级:
  1. 3 张及以上连续原子序数 → 顺序
  2. 1 张 → 单质
  3. 2 张同元素 & H/N/O/F/Cl → 单质（双原子分子 X₂）
  4. 同族 ≥2 不同元素 & skip_clan_bomb=false → 族炸
  5. 有机物检测（甲烷CH4/乙烷C2H6/丙烷C3H8及单一卤代物）→ 有机物
  6. 化合价可配平 → 化合物
  7. 否则 → 非法
```

## 5. 牌权轮转规则

顺时针轮转，N-1 人连续选择过时，牌权移交下一人。跳过的玩家抽 1 张牌。

**补充规则：顺序反转方向**——打出「顺序」牌型后，出牌方向反转（顺时针↔逆时针交替）。

---

## 6. 接牌规则

```
桌面空 → 自由出牌任意牌型
桌面单质 → 更大的单质 / 族炸
桌面化合物 → 更大的化合物 / 族炸
  例外: 溢出化合物（牌数 ≥ 玩家数）→ 免疫族炸（仍可接更大化合物）
桌面有机物 → 任何牌不可接（打出即获胜）
桌面顺序 → 无大小比较，打出后复位桌面
族炸 → 接炸链（只能出族炸）
```

---

## 7. 接炸跳过摸牌规则

**接炸中跳过现在也摸牌**：

- 族炸接炸链中，选择跳过（不接炸）的玩家同样 **抽 1 张牌** 并标记 ⏸
- 与正常跳过完全一致，包括 **上限弃牌规则** 的检查
- 手牌达到上限时，不能直接跳过，必须进入弃牌模式（选择 1 张牌弃置）
- 日志提示："XXX 不接炸，抽1张并跳过"

---

## 8. 溢出化合物规则

| 条件 | 效果 |
|------|------|
| 化合物牌数 **≥ 玩家数** | **免疫族炸**（仍可被更大化合物接牌） |
| 化合物牌数 < 玩家数 | 正常接牌（更大化合物 / 族炸） |

- 日志提示: "⚠ 溢出化合物！牌数≥N人，免疫族炸"
- 提示文本: "桌面溢出化合物(免疫族炸)，只能出更大的化合物 / 跳过"
- AI 同样受此规则约束（统一在 `play_cards()` 中生效）

---

## 9. 化合物限制规则

**卤族互化禁止**：F、Cl、Br 三种卤族元素之间不允许互相化合。

- F + Cl → ❌ "卤族元素之间不可互相化合！"
- F + Br → ❌
- Cl + Br → ❌
- Na + F → ✅ NaF（卤族 + 金属可以）
- **AI 同样遵守此规则**（通过 `_is_ai_halogen_pair()` 跳过纯卤族对）

---

## 10. 手牌上限与弃牌规则

### 上限计算

```
手牌上限 = min(玩家数 × 4, 18)
4 人局 = 16 | 5 人局 = 18 | 6+ 人局 = 18
```

### 上限弃牌规则

- 玩家手牌数 **达到或超过上限** 时，不可正常跳过（含接炸跳过）
- 必须选择恰好 1 张手牌弃置（step_mode=3）
- 弃置后标记 ⏸ 并轮转牌权，**不抽新牌**

---

## 11. 比大小规则

```
族炸 > 有机物 > 化合物 > 单质
顺序无大小比较
同牌型: 原子序数和 → 小 = 大
平局: 原子质量和 → 大 = 弱
```

**第二关：族炸进阶**（4人：1玩家+3AI）

- 步骤4：注意手牌上限规则

## 13. 游戏流程

```
Start Page → 选择模式（自由/第一关/第二关）→ 勾选补充规则
  ↓
GameManager.init_game / init_tutorial
  ├── 172 张牌洗牌 → 每人发 8 张（或预设手牌）
  └── is_round_starter = true

回合循环:
  _refresh_ui() → 牌权状态 + 桌面 + 手牌(牌面) + 牌库计数 + 教程引导
  人类操作 → 选牌 → 选牌型 → 化合物化合价 → 打出
  有机物打出 → 立即胜利
  顺序打出 → 反转方向 + 弹出指定牌型对话框
  接炸跳过 → 抽1张（上限时弃牌）
  上限弃牌 → step_mode=3 弃牌模式
  溢出化合物 → 免疫族炸（仍可接化合物）
  族炸 → 接炸链
  手牌 = 0 → GAME_OVER → End Page
```

## 16. 补充规则

补充规则可在开始页通过勾选框启用/禁用。启用后，游戏页会显示"【补充规则】有机物胜利 + 顺序牌型"提示。

### 16.1 有机物胜利规则

#### 定义

有机物在游戏中是化合物的一种特殊子类型。为简化判断，只承认以下化合物为有机物：

- **甲烷 CH₄**：C × 1 + H × 4
- **乙烷 C₂H₆**：C × 2 + H × 6
- **丙烷 C₃H₈**：C × 3 + H × 8
- **单一卤代物**：上述烷烃中一个 H 被一种卤族元素（F/Cl/Br）替换
  - 示例：CH₃Cl（C×1 + H×3 + Cl×1）
  - 示例：C₂H₅F（C×2 + H×5 + F×1）
  - 限制：**只能包含一种卤族元素**（若包含 F 和 Cl 两种则无效）

#### 打出方式

选中正确组合的牌后，在牌型选择步骤点击「合成有机物(化合物)」按钮。

#### 胜利条件

**打出有机物后，游戏立即结束，该玩家直接获胜。**

#### 实现

- 检测函数：`Utils.gd` 中的 `_is_organic()` 和 `get_organic_name()`
- 胜利判定：`GameManager.gd` 中 `play_cards()` 检测到 `ORGANIC` 类型后立即设置 `winner_index` 并结束游戏
- 牌型优先级：有机物在 `detect_pattern()` 中的检测优先级高于普通化合物，低于族炸
- 牌力比较：有机物 > 化合物 > 单质（在 `compare_cards()` 中体现）

### 16.2 顺序牌型规则

#### 定义

**三张及以上连续原子序数的牌**构成"顺序"牌型。

- 示例：H(1) + He(2) + Li(3) = 3张连续 → 顺序
- 示例：C(6) + N(7) + O(8) + F(9) = 4张连续 → 顺序
- 牌面元素顺序不要求严格按原子序数排列，系统会自动排序检测

#### 特性

一、由三张及以上连续原子序数的牌构成，**没有大小比较**

二、打出时，**出牌的顺序变更**：顺时针变为逆时针，逆时针变为顺时针

三、打出"顺序"的玩家选择让下一名玩家打出**一种特定牌型的牌**（单质/化合物/族炸/有机物），否则该玩家**罚抽两张牌**

四、无论被指定的下一名玩家出什么，他的下一名玩家都**自由出牌**（约束仅作用于紧邻的下一名玩家）

#### 打出方式

选中 3+ 张牌后，系统自动检测为"顺序"牌型。在牌型选择步骤选择任意一种牌型即可（因为顺序的判定优先于其他牌型，系统会自动识别）。

#### 弹窗指定牌型

打出顺序后，会弹出对话框让玩家选择下一名玩家必须打出的牌型：
- **单质**：下一名玩家必须打出单质
- **化合物**：下一名玩家必须打出化合物
- **族炸**：下一名玩家必须打出族炸
- **有机物**（仅在补充规则启用时可选）：下一名玩家必须打出有机物
- **跳过指定**：放弃约束权利，下一名玩家自由出牌

#### 实现

- 检测函数：`Utils.gd` 中的 `_is_sequence()` 按原子序数排序后检查是否严格连续
- 方向反转：`GameManager.gd` 中的 `direction_clockwise` 在打出 SEQUENCE 时翻转
- 牌型约束：`GameManager.gd` 中的 `set_sequence_constraint()` 和 `sequence_constraint_active` 控制
- 罚抽：`GameManager.gd` 中的 `player_fail_sequence_constraint()` 在玩家不满足约束时罚抽2张
- UI 弹窗：`GameUI.gd` 中的 `_show_sequence_constraint_dialog()` 提供选择界面
- 牌权轮转：`next_turn()` 和 `_get_next_index_in_direction()` 已支持方向切换
