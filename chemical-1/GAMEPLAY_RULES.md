## 3. 三种牌型

```
detect_pattern(cards, skip_clan_bomb=false) 判定优先级:
  1. 1 张 → 单质
  2. 2 张同元素 & H/N/O/F/Cl → 单质（双原子分子 X₂）
  3. 同族 ≥2 不同元素 & skip_clan_bomb=false → 族炸
  4. 化合价可配平 → 化合物
  5. 否则 → 非法
```

## 4. 牌权轮转规则

顺时针轮转，N-1 人连续选择过时，牌权移交下一人。跳过的玩家抽 1 张牌。（手牌达上限时除外，见 §10）

## 5. 单质

| 形式 | 牌数 | 条件 | 示例 |
|------|------|------|------|
| 单张元素 | 1 | 任意元素 | He, Na, C, Fe, K |
| 双原子分子 | 2 | 同元素且符号 ∈ {H, N, O, F, Cl} | H₂, O₂, Cl₂ |

## 6. 化合物

| 约束 | 说明 |
|------|------|
| 元素数量 | ≥2 种 |
| 2 元素 | GCD 最简比 |
| 3+ 元素 | 每种恰好 1 张 + 总电荷 = 0 |
| 金属优先 | 金属作为正价 |
| 卤族限制 | F/Cl/Br 不可互相化合 |
| 比例校验 | play_cards 中 remove_cards 之前调用 ratio_ok |

---

## 7. 族炸

| 条件 | 说明 |
|------|------|
| 元素 | ≥2 张不同元素，全部同族 |
| 等级 | 3 张 > 2 张 |
| 冷却 | 打出族炸后进入 ❄ 冷却，需打出化合物解除 |
| 接炸 | 顺时针询问各玩家是否出更大族炸 |

---

## 8. 比大小规则

```
族炸 > 化合物 > 单质
同牌型: 原子序数和 → 小 = 大
平局: 原子质量和 → 大 = 弱
```

---

## 9. 接牌规则

```
桌面空 → 自由出牌
桌面单质 → 更大的单质 / 族炸
桌面化合物 → 更大的化合物 / 族炸
  例外: 溢出化合物（见 §12）→ 免疫接牌（包括族炸）
族炸 → 接炸链（只能出族炸）
```

---

## 10. 手牌上限与弃牌规则

### 上限计算

```
手牌上限 = min(玩家数 × 4, 18)
4 人局 = 16 | 5 人局 = 18 | 6+ 人局 = 18
```

### 上限弃牌规则（新增）

- 玩家手牌数 **达到或超过上限** 时，不可正常跳过
- 点击"跳过"按钮，进入**弃牌模式**（step_mode=3）
- 玩家必须**选择恰好 1 张手牌弃置**
- 弃置后标记 ⏸ 并轮转牌权，**不抽新牌**
- 手牌降至上限以下后恢复正常操作

## 11. 化合物限制规则

**卤族互化禁止**：F、Cl、Br 三种卤族元素之间不允许互相化合。

- F + Cl → ❌ "卤族元素之间不可互相化合！"
- F + Br → ❌
- Cl + Br → ❌
- Na + F → ✅ NaF（卤族 + 金属可以）

---

## 12. 溢出化合物规则

| 条件 | 效果 |
|------|------|
| 化合物牌数 **≥ 玩家数** | **免疫所有接牌**（包括族炸！） |
| 化合物牌数 < 玩家数 | 正常接牌（更大化合物 / 族炸） |

- 日志提示: "⚠ 溢出化合物！牌数≥N人，免疫接牌（包括族炸）"
- 打出后直接轮到下家，不启动接牌流程

## 13. 游戏流程

```
Start Page → 选择人数 → Start Game
  ↓
GameManager.init_game(total, ai)
  ├── 172 张牌洗牌 → 每人发 8 张
  └── is_round_starter = true

回合循环:
  _refresh_ui() → 牌权状态 + 桌面 + 手牌 + 牌库计数
  人类操作 → 选牌 → 选牌型 → 卤族互化检查 → 化合价(化合物)
  手牌上限 → 跳过改为弃牌模式(选1张弃置)
  AI 操作 → 自动决策
  play_cards() → detect_pattern + 比大小 + 化合物比例校验
  溢出化合物(牌数≥玩家数) → 免疫接牌(含族炸)
  牌权轮转 → next_turn() → 接牌/过 → 总人数-1 连续过 → 新一轮
  族炸 → 接炸链 → 无人接 → 出炸者自由出牌
  手牌 = 0 → GAME_OVER → End Page
```
