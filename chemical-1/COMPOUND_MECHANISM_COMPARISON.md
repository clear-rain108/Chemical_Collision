## 1. 当前机制（v2严格版）

### `get_compound_formula(cards)` 流程

```
1. 统计元素出现次数 → elem_counts
2. ❌ 必须恰好 2 种元素 → 否则返回 {}
3. 分类正/负价候选：
   - 每种元素计算 max_pos / max_neg
   - 金属优先作为正价(pos_elem)
   - 找非同符号的负价(neg_elem)
4. GCD 计算最简比：ratio_pos : ratio_neg
5. ✅ 严格检查：actual_pos == ratio_pos && actual_neg == ratio_neg
6. 生成化学式
```

| 约束 | 值 |
|------|-----|
| 元素种类 | **严格 2 种** |
| 比例检查 | **精确匹配** |
| 倍数放大 | **不允许** |
| 正价选择 | **金属优先** |

## 2. 旧版机制（v1宽松版）

### `get_compound_formula(cards)` 流程

```
1. 统计元素次数 → elem_counts
2. 无元素种类限制
3. 取第一个正价 + 第一个负价
4. GCD 计算最简比
5. ✅ 宽松检查：k = actual/ratio ≥ 1 整数即可
6. 生成化学式
```

| 约束 | 值 |
|------|-----|
| 元素种类 | **不限** |
| 比例检查 | **宽松（允许倍数放大）** |
| 倍数放大 | **允许** |
| 正价选择 | **先遇到先得** |
